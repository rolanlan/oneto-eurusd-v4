## 1. Architecture Overview

### Core Insight

All five agent files (`MacroAnalyst`, `NewsAnalyst`, `PositioningAnalyst`, `RiskAnalyst`, `TechnicalAnalyst`) already have a `memoryLayer` parameter interface. `CommitteeOrchestrator` already does:

```javascript
const memory = { ...DEFAULT_MEMORY, ...memoryLayer };
```

The **only missing piece** is a service that fetches real data and passes it as `memoryLayer`. Zero agent files need modification. The entire V4.3 change is additive.

### Before V4.3

```
index.html → SignalEngine.run({ accountProfile })
              memoryLayer = {}  ← always empty
              CommitteeOrchestrator uses DEFAULT_MEMORY for all agents
```

### After V4.3

```
index.html → MemoryAggregator.getMemoryLayer()
           → SignalEngine.run({ accountProfile, memoryLayer })
              CommitteeOrchestrator merges real data into agent inputs
```

```
Purpose:   FRED API client for US/EU macroeconomic data
Key:       localStorage('fred_api_key_v4')
Base URL:  https://api.stlouisfed.org/fred/series/observations

FRED Series fetched:
  DGS10               → US 10-Year Treasury yield (daily)
  IRLTLT01DEM156N     → Germany 10-Year yield (monthly, ECB via FRED)
  A191RL1Q225SBEA     → US Real GDP QoQ % (quarterly)
  CPIAUCSL            → US CPI YoY % (monthly)
  CP0000EZ19M086NEST  → EU CPI YoY % (monthly)
  VIXCLS              → CBOE VIX closing (daily)
  UNRATE              → US Unemployment (monthly)
  LRHUTTTTEZQ156S     → EU Unemployment (monthly)

Exported:
  getMacroBundle()        → Promise<FREDBundle>
  hasApiKey()             → boolean
  testConnection()        → Promise<{valid, message}>
  clearCache()            → void

FREDBundle shape:
  {
    us10y_yield, de10y_yield, us_de_spread,
    gdp_us_qoq, gdp_eu_qoq,
    cpi_us_yoy, cpi_eu_yoy,
    vix_level, unemployment_us, unemployment_eu,
    policy_momentum, cb_fed_stance_score, cb_ecb_stance_score,
    data_source, fetched_at

```

### `src/services/FinnhubService.js`

```
Purpose:   Finnhub API — forex news sentiment + economic calendar
Key:       localStorage('finnhub_api_key_v4')
Base URL:  https://finnhub.io/api/v1

Endpoints used:
  /news?category=forex                  → news headlines
  /calendar/economic?from=&to=          → economic events

Exported:
  getNewsBundle()         → Promise<NewsBundle>
  getCalendarBundle()     → Promise<CalendarBundle>
  hasApiKey()             → boolean
  testConnection()        → Promise<{valid, message}>

NewsBundle shape:
  {
    news_net_score_24h, news_net_score_7d, news_net_score_30d,
    headline_count_24h, high_impact_count_24h,
    narrative_shift, narrative_shift_magnitude,
    dominant_theme, secondary_theme,
    data_age_hours, data_source
  }

CalendarBundle shape:
  {
    upcoming_event_risk, event_within_hours,
    event_impact_level, news_blackout,
    next_events_24h[], data_source
  }

Sentiment computation:
  score_i = headline_sentiment × exp(−age_hours / 24)
  net_24h = Σ score_i  for age_hours < 24
  Filter: headline mentions ["EUR", "USD", "ECB", "Fed", "FOMC",
          "CPI", "GDP", "NFP", "rate", "inflation"]

Calendar filter:
  currency in ['USD', 'EUR']
  impact in ['high'] for event_risk trigger
  event_within_hours = hours until next high-impact event
```

### `src/services/COTService.js`

```
Purpose:   CFTC Commitments of Traders CSV parser
Key:       None (public government data)
URL:       https://www.cftc.gov/dea/futures/deacmesf.zip
           (also available unzipped: /dea/futures/deacmesf.txt)

Strategy:
  1. Try deacmesf.txt first (no decompression needed)
  2. Fallback: deacmesf.zip + DecompressionStream API
  3. Parse CSV: find row where Market_and_Exchange_Names contains "EURO FX"
  4. Extract: NonComm_Positions_Long, NonComm_Positions_Short
  5. net = long − short
  6. Append to localStorage history (rolling 52-week array)
  7. Compute z-score vs 52-week mean/std

Exported:
  getCOTBundle()          → Promise<COTBundle>
  getHistory()            → COTWeek[]  (last 52 weeks)
  clearCache()            → void

COTBundle shape:
  {
    cot_net_position, cot_z_score_52w, cot_z_score_26w,
    cot_signal,           // 'extreme_long'|'long'|'neutral'|'short'|'extreme_short'
    cot_trend_3w,         // 'increasing'|'decreasing'|'flat'
    cot_extreme,          // |z| > 2.0
    cot_change_weekly,    // week-over-week delta
    cot_change_4week,     // 4-week cumulative delta
    data_age_days,        // days since CFTC report date
    report_date,          // ISO string
    data_source
  }

z-score signal mapping:
  z > 2.0   → 'extreme_long'   → contrarian SELL signal
  1.0–2.0   → 'long'
  −1.0–1.0  → 'neutral'
  −1.0–−2.0 → 'short'
  z < −2.0  → 'extreme_short'  → contrarian BUY signal
```

### Finnhub API

```
News endpoint:
  GET https://finnhub.io/api/v1/news?category=forex&token={key}
  Response: [{ datetime, headline, summary, source, sentiment? }]

  Sentiment scoring: keywords in headline+summary
    Bullish USD: ["Fed hawkish","rate hike","strong GDP","DXY rise",
                  "ECB dovish","EU recession","FOMC hike"]  → +10 per match
```

## 9. Degradation Strategy

```
Each field degrades independently. One API failure never blocks others.

Field fallback chain (example: vix_level):
  1. In-memory cache valid           → vix_level = 18.5  (data_source='live')
  2. localStorage cache valid        → vix_level = 18.5  (data_source='cached')
  3. localStorage cache expired      → vix_level = 18.5  (data_source='stale')
  4. No cache at all                 → vix_level = 15.0  (data_source='stub')
                                        (DEFAULT_MEMORY fallback)

MemoryAggregator merge logic:
  const result = {
    ...DEFAULT_MEMORY,       // full fallback baseline
    ...fredFields,           // overwrites matched fields if available
    ...dxyFields,            // overwrites dxy_level, dxy_trend
    ...newsFields,           // overwrites news_* fields
    ...calendarFields,       // overwrites event_* fields
    ...cotFields,            // overwrites cot_* fields
    data_source: _worstSource(allSources),
  };

data_source hierarchy (worst-case wins):
  All live    → 'live'
  Any cached  → 'cached'
  Any stale   → 'stale'
  Any stub    → 'stub'

Existing agent confidence penalties (NO CODE CHANGES needed):
  NewsAnalyst:         data_source='stub'   → confidence × 0.65
  NewsAnalyst:         data_source='cached' → confidence × 0.85 (implied)
  PositioningAnalyst:  data_source='stub'   → confidence × 0.70
  PositioningAnalyst:  data_source='cached' → confidence × 0.85 (implied)
```

## 14. Frozen Decisions

The following decisions are final and may not be changed without a new architecture revision:

| # | Decision | Rationale |
|---|----------|-----------|
| 1 | DXY source: Twelve Data ICE "DXY" symbol | Real-time market price, NOT FRED DTWEXBGS (monthly trade-weighted index) |
| 2 | Calendar API: Finnhub `/calendar/economic` | CORS-supported, same key as news, one fewer key for user |
| 3 | NO agent files modified | memoryLayer interface already exists in all agents |
| 4 | MemoryAggregator in `src/services/`, not `src/core/` | It is a data service, not an engine |
| 5 | index.html: only 2 lines change | Maximum safety, minimum disruption |
| 6 | Per-field independent degradation | One API failure must never block other fields |
| 7 | ExecutionAdapter: design only, no implementation | Pre-defines multi-broker contract without risk |
| 8 | Twelve Data DXY cache: 15min price, 4h candles | Budget: 183/800 credits/day (23%) |
| 9 | CFTC fetch: direct browser fetch, no proxy | CFTC.gov supports CORS |
| 10 | FRED: 8 series, one request each | Total: 8 credits/day, negligible |
